Module #1
Introduction

Qu'est-ce qu'un Algorithme ?

  • Un processus étape par étape qui produit une action, un peu comme une recette de cuisine !
  • Un alogorithme s'appuie sur un vocabulaire commun à tous les langages de programmation

Vocabulaire

  • Variable : endroit spécifique dans la mémoire de l’ordinateur où est stockée une valeur. La variable porte un nom et elle contien une valeur qui peut changer (ou non) au cours de l'algorithme.
  • Data type : chaine de caractère (string), nombre (int), Booléen (true/false), fonction, null, undefined, Object, etc.
  • Statement (déclaration) : une ligne de code qui définit une étape. Par exemple : x = 1
  • Condition : IF CONDITION .... ELSE IF .... ELSE ....
  • Boucle / Itération : WHILE ... DO .... // FOR ... DO ....
  • Fonction : bout de code que l'on déclare et que l'on peut ré-utiliser comme on veut.

Propriété d'un algorithme

  • Input : nous donnons des informations / datas à un alogorithmes pour qu'il puisse les traiter, faire quelque chose avec
  • Output : nous attendons un résultat de l'algorithme, que ce soit une data ou une animation visuelle
  • précision
  • correct
  • Fini
  • Généralité

Pseudo Code

  • Intérêt du pseudo code
    • Décrire comment devrait fonctionner un algorithme
    • Expliquer à des personnes non habituées aux langages de programmation
    • Designer un bout de code
  • Comment écrire du pseudo code ?
    • Syntaxe choisie
    • VAR = "hello" : un seul = pour affecter une valeur à une variable
    • VAR == "hello" : pour tester une valeur, par ex. IF ma_variable == 'hello' ALORS ....
    • IF mavariable == "hello"
        RETURN "bonjour"
      ELSE
        RETURN "Au revoir"
      ENDIF
    • WHILE mavariable > 10
        RETURN mavariable
      ENDWHILE

Garder trace d’un algorithme

  • Il s'agit de suivre étape par étape les statments d'un algorithme
  • Cela peut se faire en gardant en mémoire l'état des variables par exemple, ligne par ligne, au fil des déclarations

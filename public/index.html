théorie

  • le HTML

    - "code source de la page"
    - principe des balises ouvrante / fermante entre chervrons < et > et balises autofermantes
    - structure du document (DOM)
    - interprété par des "Agents utilisateurs" (user-agent) (navigateur principalement)
    - il s'agit d'un fichier texte avec extension .html
    -

  • modèle de document HTML5

    <!DOCTYPE html>
    <html lang="fr">
    <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <link rel="stylesheet" href="style.css">
    <script type="text/javascript" src="scripts.js" ></script>
    <title>Document</title>
    </head>
    <body>
    </body>
    </html>

  • les balises importantes

    - div, img, span, p, a

  • La sémantique HTML5 et importance sur la SEO et l'accessibilité

    - le balisage d'une page a une incidence sur le SEO du site Web
    - la sémantique des balises permet une meilleur lecture du site par les avents utilisateurs / robots
    - les attributs des balises html ont une importance dans la lecture du site par les reconnaissances.

  • de l'importance d'un code propre

    - identation du code (tous les codes)
    - structurer ses pages
    - structurer ses répertoires
    - structurer ses dépendances, etc.

  • le principe du DRY

    - Don't repeat yourself
    - gagner du temps sur les modèles : Snippets
    - gagner du temps sur les process : yarn, emmet, sass, gulp, grunt, etc.
    - penser modulaire : réutiliser un block plusieurs fois
    -
